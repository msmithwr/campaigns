# Technical Debt Register

This folder tracks deferred product and engineering work for the Campaign Command app.

Use `backlog.md` as the working register. When an item is implemented, move it to `resolved.md` with the date and a short note about what changed.

Statuses:

- `Open`: agreed debt or deferred work.
- `In progress`: currently being built.
- `Blocked`: waiting on a decision, access, vendor capability, or data.
- `Resolved`: implemented and moved to `resolved.md`.
