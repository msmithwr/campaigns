## 3. Arabic Translation Brief

### Translation Scope:
- Hero section (headline, subheadline, CTA)
- Section headlines throughout the page
- Interactive pillar section (all 6 pillar headlines and summary descriptions)
- Process section (4 steps)
- FAQ section (all Q&As)
- Booking form labels and placeholders
- CTA buttons and trust badges text

### Translation Guidelines:
1. **RTL Layout:** Full right-to-left layout implementation with mirrored navigation
2. **Cultural Adaptation:** Not direct translation — adapt messaging for Arabic-speaking professionals
3. **Formal Arabic:** Use Modern Standard Arabic (فصحى) for professional tone
4. **Technical Terms:** Keep AWS service names in English (e.g., "AWS Well-Architected Framework") with Arabic description
5. **Numbers:** Use Western Arabic numerals (1, 2, 3) not Eastern Arabic (١, ٢, ٣) for consistency with AWS documentation
6. **Font:** Use a professional Arabic web font (e.g., IBM Plex Arabic, Noto Sans Arabic) that pairs well with the English font
7. **CTA Buttons:** Translate action text (e.g., "احجز مراجعتك المجانية" for "Book Your Free Review")
8. **Form Labels:** Full Arabic translation of all form fields
9. **Toggle:** English/Arabic language toggle in header, persistent across sessions

## 4. Interactive 6-Pillar Section Design Concept

### Desktop Experience:
**Layout:** Central hexagonal diagram with 6 segments, each representing a pillar.

**Default State:**
- 6 segments displayed in a hexagonal arrangement
- Each segment shows: pillar icon + pillar name
- Subtle animation: gentle pulse/glow effect cycling through pillars
- Center of hexagon displays: "Click a pillar to explore"

**Hover State:**
- Hovered segment expands slightly (scale 1.05)
- Segment color intensifies
- Tooltip preview: pillar headline (e.g., "Run Better, Every Day")

**Active/Click State:**
- Selected segment highlighted with full color
- Other segments dim slightly (opacity 0.6)
- Content panel slides in from right with:
  - Pillar headline
  - Description text (2-3 sentences)
  - "What We Assess" bullet list (5 items)
  - "Common Findings" bullet list (4 items)
  - "Typical Improvements" with animated counters
  - Mini CTA: "Include this in my review →"
- Smooth transition between pillars when switching

**Animation Sequence (Auto-play):**
- On page load, pillars illuminate one-by-one in sequence (1 second each)
- After full cycle, rests on "Click to explore" state
- Auto-play pauses if user interacts

### Mobile Experience:
**Layout:** Stacked expandable cards (accordion-style)

**Default State:**
- 6 cards stacked vertically, each showing: icon + pillar name + arrow
- Cards have pillar-specific left border color

**Tap State:**
- Card expands smoothly to reveal full content
- Other cards remain collapsed
- Scroll position adjusts to keep expanded card in view
- Collapse animation when tapping another card

### Technical Requirements:
- **Framework:** CSS animations + vanilla JS (or lightweight library)
- **Performance:** Lazy-load pillar content, no heavy JS frameworks
- **Accessibility:** Full keyboard navigation, ARIA labels, screen reader support
- **SEO:** All pillar content in DOM (not loaded dynamically), proper heading hierarchy
